3. Delivered Changes

3.1 Exact-token anti-ambiguity matching

Problem addressed:
- Previous assistant scoring relied on substring matching in several text fields.
- This could create false positives, for example matching token "рила" inside the word "открила".

Implementation:
- Added a dedicated text matcher helper using exact token extraction and matching.
- Updated assistant scoring to use whole-token checks instead of raw substring checks.

Outcome:
- Reduced ambiguous lexical matches.
- Improved relevance precision for Bulgarian trail queries with overlapping morphemes.


3.2 Assistant provider fallback hardening (Gemini -> OpenAI)

Problem addressed:
- Existing fallback behavior was too narrow and primarily focused on Gemini 429 cases.
- It did not fully cover model unavailability or broader service-failure scenarios.

Implementation:
- Expanded fallback conditions to include unavailable model/service paths.
- Added support for secondary OpenAI fallback model configuration.
- Exposed relevant options in configuration for predictable runtime behavior.

Outcome:
- Assistant remains available when primary model path is unavailable.
- Reduced outage risk tied to single-provider/model failure modes.


3.3 schema.org JSON-LD in trail details page

Problem addressed:
- Trail details page had no explicit structured metadata for search engines and machine-readable indexing.

Implementation:
- Added JSON-LD generation and injection in the trail details view.
- Used schema.org TouristAttraction representation with geo coordinates and trail-related properties.

Outcome:
- Better structured discoverability footprint.
- Stronger semantic representation of trail entities for external crawlers and assistants.
