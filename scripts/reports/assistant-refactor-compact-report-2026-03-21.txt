EcoProject
Assistant Refactor Compact Report
Date: 2026-03-21

Goal
Progressive architecture decomposition of OpenAiAssistantService without behavior regressions.

Completed Work
1) Security & provenance decomposition
- Extracted prompt-safety policy into dedicated service.
- Extracted provenance/trust policy into dedicated service.

2) Retrieval decomposition
- Extracted context retrieval (hybrid vector + full-text ranking) and alternative recommendation into dedicated retrieval service.

3) Enrichment workflow decomposition
- Extracted enrichment orchestration loop into dedicated workflow service.
- OpenAiAssistantService now delegates enrichment execution while keeping provider-specific semantic extraction logic.

4) Provider transport extraction
- Extracted Gemini/OpenAI HTTP transport, retries and provider-level error mapping into dedicated client.
- OpenAiAssistantService now delegates provider transport calls while retaining high-level provider fallback decisions.

5) Provider fallback policy extraction
- Extracted fallback decision logic into dedicated policy service.
- OpenAiAssistantService now delegates fallback-routing decisions to policy component.

6) Weather context extraction
- Extracted weather context detection/building into dedicated weather service.
- OpenAiAssistantService now delegates weather prompt detection and weather context composition.

7) Session/chat orchestration extraction
- Extracted session get-or-create/access/title/session-query flow into dedicated orchestration service.
- OpenAiAssistantService now delegates session/chat orchestration and no longer keeps inline session helper logic.

8) Targeted unit tests for retrieval and fallback policy
- Added deterministic unit tests for retrieval ranking behavior (exact token priority + coordinate fallback ordering for alternatives).
- Added focused unit tests for provider fallback policy decisions (Gemini->OpenAI and secondary OpenAI model fallback).

9) Response composition extraction
- Extracted knowledge-chip and quick-action assembly into dedicated response composition service.
- OpenAiAssistantService now delegates response composition and no longer keeps inline chip/action builder logic.

10) Prompt assembly extraction
- Extracted assistant mode-aware prompt assembly (system + user payload builders) into dedicated prompt assembly service.
- OpenAiAssistantService now delegates prompt construction and no longer keeps inline mode/prompt builder methods.

11) Targeted unit tests for prompt assembly and response composition services
- Added focused tests for mode resolution and prompt payload composition.
- Added focused tests for knowledge-chip and quick-action composition behavior.

12) Targeted unit tests for fallback orchestration in OpenAiAssistantService
- Added focused tests for Gemini->OpenAI fallback routing and primary->secondary OpenAI model fallback routing.

13) Targeted unit tests for prompt-mode fail-open in OpenAiAssistantService
- Extended fallback-orchestration tests with `context_prompt` fail-open scenarios (exception path and empty-response path).

14) Targeted unit test for prompt-template shadow mode in OpenAiAssistantService
- Extended fallback-orchestration tests with `PromptTemplateShadowMode=true` scenario to ensure serving path remains `current` mode.

15) Targeted unit test for strict context-prompt mode in OpenAiAssistantService
- Extended fallback-orchestration tests with `PromptTemplateFailOpen=false` scenario to ensure context-prompt exceptions propagate without fallback.
