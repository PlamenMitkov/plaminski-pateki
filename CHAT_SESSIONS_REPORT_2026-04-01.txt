СЕСИЯ 7 — Trail ID duplication bug анализ (29 March 2026)
=================================================================
Дата: 2026-03-29
Session ID: 64b4fc72-135a-41bd-abef-d77ab8aae8a6 (реконструиран от base64 session memory)

Тема: Разследване на прескачане на идентификатори при import на пътеки

Проблемът:
- При reimport когато location данните в eco.json се променят (nearestTown липсва или region актуализиран), се генерира нов ред вместо update.
- SQL Server Identity seed не запълва gaps → появяват се прескачания на ID-та (напр. 1004→2004).

Root Cause анализ:
- EcoJsonImportService.cs изгражда lookup key по (Name, Location).
- Stored Location в базата може да не съвпада с recalculated Location от eco.json → lookup miss → вмъкване на нов ред.
- При изтриване на редове Identity counter не се ресетва; следващият INSERT използва текущия seed.

Идентифицирани файлове:
- EcoTrails.Api/Services/EcoJsonImportService.cs (ред ~217)

Резултат: Установена и документирана root cause; препоръка за fix чрез стабилен уникален ключ (напр. source id от eco.json вместо name+location).

СЕСИЯ 10 — SQL Server login грешка диагностика (01 April 2026, 05:20)
=================================================================
Дата: 2026-04-01
Session ID: e3ef404b-1af0-4aff-96c2-7255814deaa6

Тема: API не може да се свърже с базата данни след Docker startup

Грешка:
- "Cannot open database 'EcoTrailsDb' requested by the login. Login failed for user 'sa'."
- SQL Error Number: 4060, State:1, Class:11

Диагностика:
- docker logs plaminski-api разкриха повтарящ се login failure при EF Core query.
- Причина: stale persisted SQL Server volume от предишен run; базата EcoTrailsDb не съществував в него.

Решение:
- Извършен docker compose down -v (изтриване на volume).
- Рестарт с docker compose up --build.
- Потвърдено: containers стартирани чисто; SQL Server можа да открие EcoTrailsDb.

Резултат: Проблемът решен чрез clean compose reset; документирано в REPORT_PROGRESS.txt.
